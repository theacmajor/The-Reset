## Directory Purposes

**`.planning/codebase/`:**
- Purpose: GSD (Getting Stuff Done) analysis and planning documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, STACK.md, INTEGRATIONS.md
- Key files: All markdown documentation files generated by GSD tools

**`dist/`:**
- Purpose: Bundled and minified production build output
- Contains: Generated HTML, JS, CSS, and asset files
- Key files: `index.html` (entry point), `assets/` (minified bundles)
- Generated: Yes (by `npm run build`)
- Committed: No (excluded via .gitignore)

**`node_modules/`:**
- Purpose: Installed npm package dependencies
- Contains: React, Vite, Tailwind CSS, ESLint, PostCSS, and their transitive dependencies
- Generated: Yes (by `npm install`)
- Committed: No (excluded via .gitignore)

**`public/`:**
- Purpose: Static assets served directly by the web server without processing
- Contains: SVG, images, fonts (if any)
- Key files: `favicon.svg`
- Committed: Yes

**`src/`:**
- Purpose: Application source code
- Contains: React components, styles, entry points
- Committed: Yes

**`src/components/`:**
- Purpose: React functional components organized by feature/section
- Contains: BlueprintPage component with nested micro-components
- Key files: `BlueprintPage.jsx`
- Committed: Yes

**`src/assets/`:**
- Purpose: Imported images, SVG, and media files used by components
- Contains: Visual resources referenced in React components
- Committed: Yes

## Key File Locations

**Entry Points:**
- `index.html`: HTML document root, references `src/main.jsx` script module
- `src/main.jsx`: React DOM root creation and component mounting to `#root` element
- `src/App.jsx`: Application wrapper component, imports and renders BlueprintPage

**Configuration:**
- `vite.config.js`: Build tool configuration with React plugin
- `tailwind.config.js`: Design token definitions (fonts, colors, spacing)
- `postcss.config.js`: CSS processing pipeline (autoprefixer, Tailwind)
- `eslint.config.js`: Code quality linting rules (React hooks, React refresh)
- `package.json`: Project metadata, dependencies, build scripts

**Core Logic:**
- `src/components/BlueprintPage.jsx`: All component definitions and mock data
- `src/App.jsx`: App component wrapping and styling import
- `src/index.css`: Tailwind directives, custom utilities, print styles, font imports

**Styling:**
- `src/index.css`: Global styles, Tailwind layers, custom utilities (font families, textures)
- `tailwind.config.js`: Extended theme (serif, mono-jet fonts; ivory, ink, accent colors)

## Naming Conventions

**Files:**
- React components: `.jsx` extension, PascalCase (e.g., `BlueprintPage.jsx`, `App.jsx`)
- Configuration files: `.js` extension, kebab-case (e.g., `vite.config.js`, `tailwind.config.js`)
- Styling files: `.css` extension (e.g., `index.css`)
- HTML: `.html` extension (e.g., `index.html`)

**Directories:**
- Feature directories: lowercase, plural when multiple items (e.g., `components/`, `assets/`)
- Nested component dirs: Not used; all components in single `components/` directory

**React Components:**
- Naming: PascalCase function names (e.g., `BlueprintHeader`, `SuperpowerSection`)
- Micro-components: Descriptive names (e.g., `SectionLabel`, `DotDivider`)
- Props: camelCase (e.g., `className`)
- Constants: UPPER_SNAKE_CASE for mock data object (e.g., `data`)

**CSS Classes:**
- Tailwind utilities: lowercase with hyphens (e.g., `px-8`, `text-3xl`, `flex-col`)
- Custom utilities: lowercase with hyphens (e.g., `font-serif`, `font-mono-jet`, `paper-texture`)
- Media query prefix: `md:` for medium breakpoint and above

**JavaScript Naming:**
- Functions: camelCase (e.g., `createRoot`, `render`)
- Variables: camelCase (e.g., `data`, `strengths`, `caseStudies`)
- Objects: camelCase keys (e.g., `data.currentSalary`, `cs.primary`)

## Where to Add New Code

**New Section Component:**
- Primary code: Add function to `src/components/BlueprintPage.jsx` following pattern of existing sections (e.g., `SuperpowerSection`, `CaseStudySection`)
- Styling: Use Tailwind classes inline in JSX; add custom utilities to `src/index.css` if needed
- Data: Extend the `data` constant in BlueprintPage with new properties
- Integration: Add section render call in `BlueprintPage()` JSX output in proper document order

**New Micro-component (reusable UI element):**
- Location: Define in `src/components/BlueprintPage.jsx` above usage (following `SectionLabel`, `Divider`, etc. pattern)
- Accept optional `className` prop for flexible styling
- Return minimal JSX with Tailwind utilities

**New Utility Class or Custom CSS:**
- Location: `src/index.css` in appropriate `@layer` directive (base, components, or utilities)
- Example: Custom font families defined in `@layer utilities` (see `font-serif`, `font-mono-jet`)
- Reference in components via class names

**New Static Asset:**
- Location: Place in `src/assets/` directory
- Reference: Import and use in React components (e.g., `import logo from './assets/logo.svg'`)

**New Configuration:**
- Tailwind theme extension: Add to `tailwind.config.js` theme.extend object
- PostCSS processing: Modify `postcss.config.js` if needed
- Vite build: Modify `vite.config.js` if adding plugins or build options
- ESLint rules: Modify `eslint.config.js` for new code quality checks

**New Page/Route:**
- Limitation: Application currently has no routing library (React Router)
- To add: Install react-router-dom, configure routing in `src/App.jsx`, create new page component in `src/pages/` directory

## Special Directories

**`dist/`:**
- Purpose: Output directory for production build artifacts
- Contains: Bundled HTML, minified JavaScript and CSS, processed assets
- Generated: Yes (by running `npm run build`)
- Committed: No
- Regeneration: Safe to delete; will be regenerated by build process

**`.planning/codebase/`:**
- Purpose: GSD documentation for architecture, conventions, tech stack, concerns analysis
- Contains: Markdown analysis documents
- Generated: Yes (by GSD mapping tools)
- Committed: Yes (these become part of planning infrastructure)

**`node_modules/`:**
- Purpose: Local node package installation directory
- Contains: All npm dependencies and transitive dependencies
- Generated: Yes (by `npm install`)
- Committed: No (excluded in .gitignore)
- Regeneration: Run `npm install` to restore from package-lock.json
